Practice Exercise: Subnetting a Class C IP Address

Introduction

In this lab, you will assume the role of a system administrator learning the fundamentals of subnetting a Class C IP address. Subnetting is crucial for efficiently managing IP addresses in a network. You will use a real-world scenario and step-by-step procedures to practice subnetting and understand its significance in computer networking.

Prerequisites

  • Basic understanding of IP addresses and binary conversion
  • Pen and paper or a calculator for manual calculations

Lab Scenario

You are responsible for setting up a network for a small company that has been assigned a Class C IP address block. The company needs to create multiple subnets to accommodate various departments. Your task is to subnet the Class C IP address effectively, ensuring optimal use of IP addresses while meeting the company's requirements. Solutions are provided at the end of the document, but make sure to understand each task.

Tasks

Task 1: Determine Requirements

  • Gather information about the company's network requirements, including the number of subnets needed and the number of hosts per subnet.
  • Calculate the total number of subnets required.

Task 2: Choose Subnet Mask

  • Decide on an appropriate subnet mask based on the number of required subnets and hosts per subnet.
  • Calculate the subnet mask in both dotted-decimal and binary notation.

Task 3: Determine Subnet Size

  • Calculate the number of bits required for the subnet portion of the IP address.
  • Determine the subnet size (number of IP addresses in each subnet).

Task 4: Create Subnet Table

  • Create a subnet table that lists each subnet's network address, range of usable IP addresses, and broadcast address.
  • Calculate these values for each subnet.

Task 5: Assign Subnets

  • Assign subnets to different departments or segments of the company based on their specific needs.
  • Ensure that each department has its own subnet and range of IP addresses.

Task 6: Troubleshooting Subnet Configuration

  • Simulate an incorrect subnet configuration by assigning overlapping IP address ranges to different subnets.
  • Identify the conflict and troubleshoot by reconfiguring the subnets correctly.

Task 7: Verify Subnet Connectivity

  • Configure two devices within the same subnet.
  • Use the ping command to verify connectivity between the devices.
  • Ensure that they can communicate successfully.

Task 8: Calculate Available Hosts

  • Determine the number of available host addresses in each subnet.
  • Calculate the broadcast address for each subnet.

Task 9: Troubleshoot Subnet Size

  • Simulate a scenario where the subnet size is too small to accommodate the required number of hosts.
  • Troubleshoot the issue and adjust the subnet size accordingly.

Task 10: Reallocation of Subnets

  • Modify the subnetting scheme to allocate more IP addresses to a specific department that is expanding.
  • Update the subnet table and configuration accordingly.

Sample Troubleshooting Command

  • Simulate an incorrect subnet configuration by assigning overlapping IP address ranges to different subnets.
  • Use the subnet table to identify and troubleshoot the conflict.

Solutions

Task 1: Determine Requirements

  • The company requires 8 subnets to accommodate different departments.

Task 2: Choose Subnet Mask

  • Based on 8 subnets, a subnet mask of /29 is chosen (255.255.255.248 in dotted-decimal notation).
  • In binary notation, the subnet mask is 11111111.11111111.11111111.11111000.

Task 3: Determine Subnet Size

  • 3 bits are required for subnetting (2^3 = 8).
  • The subnet size is 8 IP addresses per subnet.

Task 4: Create Subnet Table

  • Subnet 1: Network Address: 192.168.1.0, Usable Range: 192.168.1.1 to 192.168.1.6, Broadcast Address: 192.168.1.7
  • Subnet 2: Network Address: 192.168.1.8, Usable Range: 192.168.1.9 to 192.168.1.14, Broadcast Address: 192.168.1.15
  • (Repeat for all 8 subnets)

Task 5: Assign Subnets

  • Subnet 1: HR Department
  • Subnet 2: Sales Department
  • (Assign departments to subnets as needed)

Task 6: Troubleshooting Subnet Configuration

  • Identify the conflict in subnet 2 and resolve it by adjusting the IP address range.

Task 7: Verify Subnet Connectivity

  • Configure devices within the same subnet (e.g., 192.168.1.2 and 192.168.1.3).
  • Use ping to verify connectivity.

Task 8: Calculate Available Hosts

  • Each subnet has 6 available host addresses.
  • Broadcast address for subnet 1: 192.168.1.7

Task 9: Troubleshoot Subnet Size

  • Increase the subnet size to /28 (16 IP addresses) for the expanding department.

Task 10: Reallocation of Subnets

  • Reallocate IP addresses to the expanding department's subnet.
  • Update the subnet table and configuration accordingly.

These exercises will help you understand the process of subnetting a Class C IP address and its practical applications in network management.
